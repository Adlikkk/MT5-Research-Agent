# UI Roadmap

> **Status:** the desktop UI is now implemented — see [DESKTOP_UI.md](DESKTOP_UI.md)
> and [`ui/`](../ui). This file is kept as the original design rationale.

UI work starts only after the CLI background runner produces valid reports reliably.

## Phase UI-1

Local read-only dashboard for tasks, run status, logs, and parsed reports.

## Phase UI-2

Chat or request composer for writing research requests and smoke tasks.

## Phase UI-3

Results, leaderboard, and run-history tabs.

## Phase UI-4

Packaged desktop release around the deterministic local CLI core.
